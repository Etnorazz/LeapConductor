This article describes the Leap application that acts as an intermediary between the Leap device and your Leap-enabled application.

Overview

The Leap application communicates with the Leap device and transmits hand tracking data to Leap-enabled applications. Thus, the Leap application must be running when you access Leap data in your application.

When the Leap application is running, it displays an icon in the notification area of the Windows Taskbar or Mac Finder bar. When the Leap device is plugged in and working properly, the icon turns green. The icon will flash yellow while the software is initializing and turn red if an error has occurred.

When you first run the Leap software, it checks to see if you have logged in with your Leap developer credentials and whether you have the calibration file for your Leap device, which is necessary to run the Leap. The Leap software automatically downloads the calibration file for your device after you log in.

The Leap application has a number of commands that modify how tracking data is produced and which give you information about the state of the device. Access these commands through the menu attached to the application icon.

Note that these commands are targeted toward developers. The consumer version of the Leap application will have a different set of user commands and options.

Operating modes

The Leap has several operating modes which determine how the application analyzes data. The following modes are user-selectable from the Quality menu of the Leap Application:

  • Precision â€” prioritizes precision over speed
  • Balanced â€” balances precision with speed
  • High Speed â€” prioritizes speed over precision
  • Low Resource â€” reduces the USB bandwidth used by the Leap

The Leap also has a robust mode that improves the reliability of the tracking data in bright lighting conditions. Robust mode allows the Leap to operate in a wider range of environmental conditions; however, other performance characteristics may be reduced. The main effects on performance are that there will be increased processing latency and very fast motions by the user will cause a loss of tracking data.

The Leap automatically drops into robust mode when lighting conditions become poor. The Leap switches back into the last user-selected operating mode when lighting conditions improve (but will remain in robust mode for at least 30 seconds). The Leap application icon changes from green to red when in this mode and the quality menu is temporarily disabled.

Diagnostics

If you are having problems with the Leap software, please contact us so we can help you resolve the issue. We may ask you to reproduce the issue on your device and record some diagnostic information which you can send to Leap Motion to help us assess the problem. You can start and stop the recording of diagnostic information from the Leap application menu (on the Windows Taskbar or Mac Finder bar). The data is compressed and encrypted and is not useful for your local application debugging. The data recorded can grow large over time, so we do not recommend recording diagnostics for an extended period.

To record diagnostics,

  1. Select Start Diagnostics from the Leap application menu.
  2. Reproduce the problem (it may help to have the Leap Visualizer running).
  3. Select Stop Diagnostics after a short period of time.

The diagnostic files can be found in the following locations:

  • On Mac: ~/Library/Application Support/Leap Motion
  • On Windows: C:\Users\<user_name>\AppData\Roaming\Leap Motion

Note: These directories may be hidden by default.

Application log

To view events related to the Leap application, open the application log using the Leap application menu from the Windows Taskbar or Mac Finder bar. If you are having problems, we may ask you to copy the contents of this log and send it to Leap Motion to help us assess the problem.
